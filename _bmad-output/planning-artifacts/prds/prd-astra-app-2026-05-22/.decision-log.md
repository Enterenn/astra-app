## Consistency correction pass (2026-05-22)

| Topic | Decision |
|-------|----------|
| Purge semantics | Phase 0 purge deletes all Timeseries Samples and derived collection state, while preserving non-health setup preferences (`daily_step_goal`, onboarding, permission choices) |
| DataIngestionSource contract | Sources emit raw platform `StepReading` events; `StepNormalizer` alone creates storage-ready step buckets |
| Benchmark scope | KPI-01 / NFR-1 benchmark uses 90 days of injected data |
| Maintenance on iOS | iOS `VACUUM` is opportunistic after foreground/resume or best-effort BGAppRefresh; no Phase 0 acceptance criterion depends on reliable iOS background maintenance |
| Step value validation | `timeseries_samples.value` remains `REAL` for future series but requires non-negative whole counts for `type=steps` |
| CSV/share contract | Export preserves `id`, writes a local temp/cache file before share sheet, and treats third-party sharing as user-initiated outside ASTRA's health pipeline |
| Phase wording | Phase 0 is phone-only implementation; wearable/BLE/sync remain future context unless explicitly marked in scope |
